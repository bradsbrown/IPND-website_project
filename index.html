Stage 0

Background
The web works via a server sending html via the internet to your computer's browser, which uses the encoded text to format a web page.
The html can include various tags and elements, which are encoded information the computer uses to ehance the text with formatting, images, etc.
Inline vs. block
Tags can be either inline or block, which means they either stay in the flow of the text, or they basically "draw a box" around a piece of content.

HTML
  • HTML works in a tree-like structure
  • Chrome Dev Tools will show the HTML for a page
  • it highlights the relevant section on the page
  • it will also sync with the style elements of that section
  • All HTML elements are rectangles
  • Element are labeled with class names for css to describe formats
  • "boxing" helps lay out the positioning of elements on the page
CSS
  • Lead a class description with .classname, then include style in braces.
  • You can define a class as "*" (no leading period) to describe default style elements for the page.
  • For standard tags such as "b", "en", "h1", you can denote custom styling in the same format (no leading period)
  • Styles from the most specific level will override those up the higherarchy.
  • This is a great site for css reference.

Section 2

Introduction to Python
  • Python is a coding language, with a specific syntax that must be followed to prevent errors.
  • variables are defines using the syntax [variable name] = [number]
  • Strings are defined in the same way, but require quote mark (single or double) to indicate they are treated as text strings rather than integers or floats.
  • While you can do standard math with variables, using a '+' on strings will concatenate them, or using a '*' will repeat it.
  • You can use indexing to return certain positions of a string - string[0:3] would give you the first 3 characters (starting at position 0,ending at position 3 exclusive)
  • string.find('text_to_find') will return the starting position of the text to find
  • you can also give .find a second variable to have it start searching at that position in the string - string.find('text_to_find',5) will start looking at position 5
Functions and Loops
  • a function is a block of code that executes when called, and can take parameters to pass into the code.
  • Any number of parameters can be passed in, but the number must be defined in the def for the function, and the quantity passed in when called must match.
  • A test evaluates an expression, and returns True or False. <, >, ==, != are common tests.
  • An 'if' statement operates code if a test is met, and can also have an 'else' block to run if the test returns False.
  • Where an 'if' statement only passes through once, a 'while' loop runs reapeatedly as long as the test is True.
  • A 'break' statement kicks the code out of the loop, and continues to run the code after the loop.
Debugging
  • Googling for error messages will help find others who have solved this problem.
  • You can also compare to working code from others.
  • Sometimes dropping some print commands for intermediary variables/strings will help you see what your code is doing, and correct some wrong assumptions you may have.
  • Some form of versioning is important to be able to look back at old code and see where you went wrong. This can be simply saving files, or using a more comprehensive system like Git.
  • Commenting your code well will be a huge help when you go back to look at your work, especially if you haven't worked on a section in a while.
Lists
  • Lists are a conceptual extension of strings in a way, except that where a string is a 'list' of characters, a list can contain anything - strings, values, other lists, etc.
  • You can access values of a list by indexing using brackets, just as you could access a character in a string.
  • Aliasing links another variable name to the same list, so any changes made to a value via one alias will affect all variable names tied to that list.
  • '.append' allows you to add new items to the end of a list, len() gives you the top-level count of entries in a list, '+' concatenates items to a list.
  • If you append a list to a list, the appended list will be appended as a single entry, whereas '+'-ing the two lists will create a new list with all entries of the two.
  • 'for item in list' is a valid syntax that loops the code below it for all items in a list.
  • 'in' and 'not in' can also be used as tests that return True or False (useful for if statements).
